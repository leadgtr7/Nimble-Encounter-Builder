Nimble Encounter Builder - User Guide

Getting Started

Nimble Encounter Builder is a combat management tool designed for tabletop RPG sessions. It helps you track initiative, hit points, conditions, and more for both monsters and heroes.

Quick Start: On first launch, select a config location. If you skip, the app prompts for a base folder and creates data/Encounters, data/Heroes, and data/Combat Logs. After that, set your monster vault path in Config and click Scan Vault in Bestiary.

Combat Tab

The main combat tracking interface where you manage active combatants during encounters.

Table Columns

Column Description
Act Checkbox to mark if the combatant is currently active in combat.
Heal Heal button - click to apply healing to this combatant.
Name Combatant's name. Click to select and view stat block in the right panel.
Hurt Hurt button - click to apply damage to this combatant.
Mk Map marker - shows marker number with colored background for tracking on battle maps.
Con Concentrating checkbox - check when the combatant is concentrating on a spell or effect.
Conds Displays active conditions as colored tags (e.g., Blinded, Stunned, Prone).
HP Current hit points. Color-coded: Healthy, Bloodied, Critical, Dead/Dying
Tmp Temporary hit points. These are lost first before regular HP.
Max Maximum hit points for this combatant.

Combat Controls

Add Combatant: Manually add a monster or NPC to the combat.
Delete Selected: Remove selected combatants from combat. Hold Ctrl to select multiple rows.
Clear All: Remove all combatants from the combat table (does not affect saved encounters).
Set Color: Assign map markers to selected combatants. Select multiple monsters (hold Ctrl and click) and assign them the same color with sequential numbers.
Reset Round: Clears all damage, conditions, and temporary HP from all combatants.

HP Status Colors

Status HP Range Description
Healthy > Bloodied Threshold Green HP text - combatant is at full or high health
Bloodied >= Bloodied % but > Critical % Orange HP text - combatant is injured but still fighting
Critical/Last Stand >= Critical % but > 0 Red HP text - combatant is in critical danger
Dying/Dead 0 HP Purple Conds field - combatant is unconscious, dying, or dead
Tip: You can configure HP thresholds in the Config tab under "HP Thresholds". Set different values for monsters and heroes.

Stat Block Preview

Click any combatant in the table to view their full stat block in the right panel. This is a handy reference for abilities, attacks, and special features during combat.

Loot Display

The bottom panel shows loot drops from defeated monsters. You can click on combatants at 0 HP to view their loot information.

Heroes Tab

Manage your party of player characters.

Hero Management

Add Hero: Create a new hero with name, player name, level, and HP.
Edit Hero: Double-click a hero's name to edit their details.
Delete Hero: Remove selected heroes from the party.

Party File Management

Load Party: Load a saved party file (JSON format). Heroes are added to combat automatically.
Save Party: Save current heroes to a JSON file for reuse in future sessions.
Tip: Save your party file once and reload it at the start of each session. Hero HP and conditions can be reset as needed.

Bestiary Tab

Build and manage encounters using monsters from your vault.

Monster Vault

The Monster Vault is your library of monster templates. These are loaded from JSON files containing monster stat blocks.

Scan Vault: Load monsters from the vault path specified in Config. The path can include multiple JSON files (multi-select) and will merge them into one library.

Building Encounters

Adding Monsters: Select monsters from the vault list and click "Add to Encounter". Monsters are automatically added to combat when you add them to the encounter.
Edit Monster: Double-click a monster's name in the encounter list to edit its stats. Changes only affect this instance.
Remove from Encounter: Select monsters and click "Remove" to delete them from the encounter.
Double-Click Functions: Double-click a monster's name to edit details, or double-click the level field to quickly adjust monster level (supports fractional levels).

Encounter File Management

New Encounter: Clear the current encounter to start fresh.
Load Encounter: Load a previously saved encounter file (JSON format).
Save Encounter: Save the current encounter for future use.
Auto-Refresh Feature: When loading an encounter, you'll be prompted to refresh monster data from the vault. This updates monsters with the latest stats from your library. Enable "Auto-refresh" in Config to skip this prompt.

Encounter Difficulty

The encounter difficulty label appears at the bottom of the Bestiary tab and is calculated based on the total encounter strength compared to your party. The difficulty is color-coded to show the challenge level:

  • Easy - Green - The party should handle this without much difficulty
  • Medium - Yellow/Orange - A moderate challenge with some risk
  • Hard - Orange - A tough fight that will test the party
  • Deadly - Red - Very dangerous, potential for character death
  • Very Deadly - Purple - Extremely dangerous encounter

The difficulty calculations are based on monster levels and party composition, using thresholds you can customize in the Config tab.

Config Tab

Customize the application settings and appearance.

HP Thresholds

Set when combatants are considered Bloodied or in Critical condition as a percentage of max HP.

  • Monster Bloodied: Default 50% - monsters show orange background
  • Hero Bloodied: Default 50% - heroes show orange background
  • Monster Critical: Default 25% - red background warning
  • Hero Critical: Default 25% - red background warning

Map Markers

Color Palette: Visual list of available marker colors. These are used to identify monsters on battle maps.
Add Color: Click the color preview box to open a color picker, then click "Add to Palette" to add custom colors.
Starting Number: First number to use when auto-assigning markers (default: 1).

Logging Options

Control what events are logged to the combat log:

  • Log Damage: Record when damage is dealt
  • Log Healing: Record healing events
  • Log Deaths: Record when combatants reach 0 HP
  • Log Conditions: Record condition changes
  • Log Last Stand: Record when combatants enter critical HP range

Autosave

Enable Autosave: Automatically save combat state periodically.
Autosave Path: Location where autosave files are stored.

Paths & Files

Monster Vault Path: Location of your monster library JSON files. Use multi-select to add more than one file.
Obsidian Vault Path: Set the root directory of your Obsidian vault. When configured, all folder paths become relative to this vault, making it easy to organize your campaign data in one location.
Encounter Folder: Default location for saving/loading encounters. Can be absolute or relative to the Obsidian vault.
Party Folder: Default location for saving/loading party files. Can be absolute or relative to the Obsidian vault.
Combat Log Folder: Location where combat logs are saved. Can be absolute or relative to the Obsidian vault.
Config File: Current configuration file being used. The path is saved in config_location.json in the app folder.
Obsidian Integration: If you use Obsidian for campaign notes, setting the vault path allows you to keep all your encounter files, party data, and combat logs alongside your notes. See OBSIDIAN_VAULT_INTEGRATION.md for detailed setup instructions.

Monster Vault Options

Auto-refresh on Encounter Load: When checked, automatically updates monster stats from vault when loading encounters without prompting.

Table Appearance

Customize column widths and font sizes for the Heroes and Combat tables to fit your screen.

Saving Configuration

Save Config: Quick save to the current config file. A timestamp message confirms the save.
Save As: Save configuration to a new file (useful for multiple campaigns).
Load Config: Load a previously saved configuration file.
Refresh UI: Reload the interface with current settings (use after changing column widths or fonts).
Tip: The green status message below the buttons shows when config was last saved with a timestamp.

Vault Viewer Tab

Scan folders of markdown and JSON to build monster libraries from your vault or notes.

Add Folder: Add multiple input folders (useful when your vault content lives in different subfolders).
Scan: Parse the selected folders and build a monster list.
Export: Save the scanned monsters to JSON for use in Bestiary.

Conditions System

Nimble uses D&D 5e-style conditions to track status effects on combatants.

Available Conditions

Condition Effect
Blinded Can't see. Attacks against have advantage, its attacks have disadvantage.
Charmed Sees charmer as an ally. Charmer has advantage on social interactions.
Dazed Heroes lose one action; monsters can perform one less action.
Deafened Can't hear and automatically fails hearing-based checks.
Frightened Disadvantage when source of fear is nearby; speed halved moving toward source.
Grappled Speed becomes 0. Attacks against have advantage.
Hampered Movement or actions are reduced (general debuff category).
Incapacitated Can't take actions or reactions. Attacks against have advantage.
Invisible Can't be seen. Attacks against have disadvantage, its attacks have advantage.
Paralyzed Incapacitated, can't move/speak, fails STR/DEX saves. Attacks have advantage and crit within 5ft.
Petrified Turned to stone. Immune to most damage except massive force.
Poisoned Disadvantage on attack rolls and ability checks.
Prone Movement costs double. Disadvantage on attacks. Melee attacks against have advantage.
Restrained Speed 0. Attacks against have advantage, its attacks have disadvantage, disadvantage on DEX saves.
Riding Moves with mount. Missed attacks strike the mount.
Slowed Speed halved during next turn.
Stunned Incapacitated, can't move, barely speaks. Fails STR/DEX saves. Attacks against have advantage.
Taunted Disadvantage on attacks except against the taunter.
Unconscious Incapacitated, unaware, drops items, falls prone. Fails STR/DEX saves. Attacks crit within 5ft.
Note: HP-based statuses (Bloodied, Dying, Last Stand) are automatically tracked and do not appear in the conditions list.

Map Markers

Map markers help you track which monster is which on your battle map or VTT.

Assigning Markers

  1. Select one or more monsters in the Combat table (hold Ctrl to select multiple)
  2. Click the "Set Color" button
  3. Choose a color from the palette
  4. Set the starting number
  5. Click OK - monsters will be assigned sequential numbers with that color

Marker Display

In the "Mk" column, you'll see the marker number with a colored background. For example:

  • Red 1, Red 2, Red 3 for three goblins
  • Blue 1, Blue 2 for two orcs
  • Green 1 for the orc chief

Auto-Suggest Numbers

When assigning markers, the dialog automatically suggests the next available number for the selected color, preventing duplicates.

Workflow Tips

Starting a Session

  1. Load your party file from the Heroes tab
  2. Go to Bestiary tab and load your prepared encounter (or build one from the vault)
  3. Monsters are automatically added to the Combat tab when you add them to an encounter
  4. Assign map markers to monsters (optional but recommended)
  5. Begin combat tracking!

During Combat

  1. Click "Hurt" button to apply damage to a combatant
  2. Click "Heal" button to apply healing to a combatant
  3. Check "Act" boxes to mark combatants as active in combat
  4. Use "Con" checkbox to mark when a combatant is concentrating on a spell or effect
  5. Click on a combatant's name to view their stat block
  6. Use "Reset Round" to clear all damage, conditions, and temporary HP when combat ends

Preparing Encounters

  1. Scan your Monster Vault to load all available monsters
  2. Build encounters in the Bestiary tab by adding monsters
  3. Save encounters with descriptive names (e.g., "Goblin Ambush", "Dragon Lair")
  4. Reload saved encounters for quick setup during sessions
Pro Tip: Create a separate config file for each campaign. This keeps your monster vaults, encounter folders, and party folders organized per campaign.

File Formats

Monster Vault JSON

Monster library files use JSON format with fields like: name, level, hp, armor_class, abilities, attacks, etc. See the shared_statblock module for the complete schema.

Encounter Files

Saved encounters include the list of monsters, their HP, conditions, markers, and any instance-specific modifications.

Party Files

Party files store hero names, player names, levels, max HP, and current state. Load at session start to restore your party.

Config Files

Configuration is stored in JSON format with all your preferences: thresholds, paths, colors, logging options, and UI settings.

Keyboard Shortcuts

  • Ctrl + Click - Select multiple table rows
  • Double-Click Name - Edit hero or monster details
  • Click Name - View stat block in preview panel

Troubleshooting

Monster Vault Not Loading:
  • Check that the vault path in Config points to JSON files
  • Use multi-select if your vault is split across files
  • Click "Scan Vault" in the Bestiary tab after setting the path
  • Verify JSON files are properly formatted
Conditions Not Appearing:
  • Make sure "Log Conditions" is enabled in Config if you want them logged
  • Conditions appear as colored tags in the "Conds" column
  • Click the "Con" checkbox to open the conditions dialog
Config Not Saving:
  • Use "Save Config" button (not just checking boxes)
  • Look for the green "Saved successfully" message with timestamp
  • Some settings (like auto-refresh checkbox) auto-save immediately
Table Columns Too Small:
  • Go to Config tab and adjust column widths under "Table Appearance"
  • Click "Save Config" then "Refresh UI" to apply changes
